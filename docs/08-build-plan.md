## Sequencing note ‚Äî why build order ‚âÝ runtime order

The runtime pipeline runs intent ‚Üí query ‚Üí answer ([04-architecture.md](04-architecture.md)). The **build** order deliberately differs: we build the deterministic core (query / validation / derivations) **before** the LLM intent parser. Reasons (decided 2026-07-03): the deterministic layer is the anti-hallucination core, it needs no LLM or API key so it is fully testable in hermetic CI, and it lets us score B1‚ÄìB14 against the frozen answer key with zero LLM involved ‚Äî so when the parser lands next it targets a *known-good* query layer and a *fixed* intent contract, and any failure is unambiguously the parser's.
