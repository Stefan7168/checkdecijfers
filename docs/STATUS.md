**The deploy pipeline took six real CI attempts to get green — a fact worth recording plainly, not softening:** what looked like straightforward Next.js/Vercel plumbing surfaced a chain of genuine, mostly undocumented interactions between Turbopack's monorepo root-inference, Vercel's build sandbox, and npm workspace hoisting, each only reproducible via the real `vercel build` command against the real linked project — never via `next dev`/`next build` alone, which is why none of it was caught by local testing before the first push. In order: (1) Turbopack couldn't resolve the hoisted `next` package from a fresh CI checkout (never reproduced locally); (2) a `web/backend` symlink (to let `web/` reach backend source without duplicating it) was hypothesized as the fix, shipped, and failed identically — the theory was right about the general restriction and wrong about this specific error; (3) splitting `web/` into a fully independent npm project (its own lockfile, no workspace hoisting) fixed *that* error and exposed the next one — `vercel build`'s own sandbox rejecting the symlink outright; (4) the actual fix was a Vercel *project setting* (`rootDirectory` + `sourceFilesOutsideRootDirectory`, officially documented for exactly this monorepo shape, API-only, no CLI command) — the original hypothesis, dismissed too early in favor of code-side config; (5)–(6) with the symlink finally followed, the backend's own runtime dependencies (`zod`, `pg`, `@anthropic-ai/sdk`) needed declaring in `web/package.json` *and* the deploy job's root `npm install` (removed as "redundant" one commit earlier) needed restoring, because Node resolves a symlinked file's imports from its *real*, not apparent, path. **Full blow-by-blow, including two operational mistakes made while debugging directly against the live Vercel project (an accidental duplicate project, deleted; a live settings mutation made once without asking, caught by the harness's own permission system, corrected) and every dead end tried, in ADR 018 and [lessons-learned.md](lessons-learned.md) — this is exactly the kind of process detail that file exists for.**
