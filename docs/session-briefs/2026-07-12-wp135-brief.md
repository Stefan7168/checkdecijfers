## 1. Migration `migrations/019_chat_threads.sql` (hermetic file only â€” no live DDL)

```sql
-- chat_threads stores WHEN, never WHAT: no text columns, by design (ADR 033 D1).
-- No GRANT/RLS statements needed: migration 003's defaults lock this table (011/018 posture).
create table chat_threads (
  id bigint generated always as identity primary key,
  user_id uuid not null,
  created_at timestamptz not null default now(),
  last_activity_at timestamptz not null default now()
);
-- guarded FK to auth.users, conditional on auth schema existence (migration 005 pattern)
create index chat_threads_by_user_activity on chat_threads (user_id, last_activity_at desc);

alter table audit_answers add column thread_id bigint references chat_threads(id);
-- no ON DELETE clause: redact-not-delete posture (migration 005 precedent)
create index audit_answers_by_thread on audit_answers (thread_id) where thread_id is not null;
```

## 2. Backend: `web/backend/threads/index.ts` (new, mirrors `billing/history.ts` conventions)

All SQL with **bound parameters**; every query scoped `user_id = $n`. Functions:

- `ensureThread(db, userId, rawThreadId)` â†’ validated thread id. If `rawThreadId` is null,
  malformed, or **not owned by userId** â†’ create a fresh thread (fail-safe, the ADR-021
  invalid-context treatment; a forged id NEVER attaches rows to another user's thread).
- `attachAuditRow(db, userId, threadId, auditId)` â†’ `UPDATE audit_answers SET thread_id = $1
  WHERE id = $2 AND user_id = $3 AND thread_id IS NULL` + touch `last_activity_at`.
  **Deliberate design: the answer pipeline (`src/answer/**`) is NOT touched** â€” thread attach is
  a post-hoc UPDATE after the pipeline returns (UPDATE on audit rows is precedented by
  redaction). If the attach fails, the answer stands threadless (degraded, logged) â€” never block
  or roll back an already-audited answer.
- `listThreads(db, userId, limit=50)` â†’ `{id, title, lastActivityAt}[]`. Title = first
  **non-redacted** row's `question` (`question <> REDACTED_QUESTION_TEXT`, order created_at asc,
  id asc), truncated to 60 chars in TS. Threads whose every row is redacted (title source NULL)
  are **filtered out** â€” self-service deletion and the 2-year purge empty the sidebar with zero
  new machinery (ADR 033 D2). Import the sentinel from `src/answer/audit/retention.ts` â€” never
  duplicate the string.
- `getThreadRows(db, userId, threadId)` â†’ the thread's audit rows (created_at asc, id asc):
  `id, kind, question, final_text, reply_text, created_at, response` (full envelope) â€” same
  zero-loss posture as `history.ts`.

## 3. Replay + context rebuild (deterministic, zero LLM)

New `web/backend/threads/replay.ts`:

- `replayMessages(rows)` â†’ the `ChatMessage[]` shape `chat.tsx` renders. Per row: a user message
  from `question` (and the reply turn from `reply_text` when present), then the assistant message
  from the stored envelope: `final_text` as text, `answerView` parts from `response.answer`
  (only when the structural fields are present â€” the #115 zero-loss rule, else the `final_text`
  blob), `chart`, `suggestions`, `webSection`, `provisional`, `auditId = row.id` (feedback
  buttons work on replayed answers). **R8 pin: replayed message text is byte-equal to stored
  `final_text`.** Redacted rows replay as the existing "verwijderde vraag" placeholder posture.
- `rebuildContext(rows)` â†’ run the existing `buildConversationContext` (src/answer/context/
  build.ts) over the LAST envelope that yields non-null, walking backwards; then
  `validateConversationContext` against the live registry. Any parse/validation failure â†’ null
  (standalone parse â€” honest degradation, ADR 021). No new context logic is written; only the
  existing deterministic functions are called.

## 6. Test pins (the gate; add to existing suites, hermetic PGlite)

1. **Cross-user isolation:** `listThreads`/`getThreadRows`/`loadMyThread` return nothing for
   another user's thread; forged `rawThreadId` on ask/reply NEVER attaches a row to another
   user's thread (fresh thread instead). Mirrors the #14 cross-user pins.
2. **R8 replay:** replayed text byte-equal to stored `final_text`; answerView parts only when
   structural fields present (zero-loss rule).
3. **Context rebuild determinism:** equals `buildConversationContext` over the last eligible
   envelope; unparseable/stale envelope â†’ null; validation failure â†’ null.
4. **Redaction:** fully-redacted thread vanishes from `listThreads`; partially redacted thread
   titles from the first non-redacted row; existing retention suite untouched and green.
5. **Lazy creation:** no thread row until a first completed question; "nieuwe chat" alone
   creates nothing; `last_activity_at` touched on ask AND clarification reply.
6. **Dormancy:** flag off â†’ Dashboard render byte-identical (existing tests + a flag-off pin);
   no thread action reachable.
7. **Envelope byte-identity:** rendering test that the message envelope block (incl. web section
   last-in-bubble) is identical in dock mode vs inline mode vs today.
8. **Benchmark neutrality:** runner path sends no threadId; benchmark rows `thread_id` NULL.
9. **Shell:** signOut action; header presence rules (stripped on `/login`, present on `/`,
   `/credits`, `/geschiedenis`); footer static content EXACT string (byte-pinned; owner gives
   the final look at PR review).
10. **Migration 019** runs green in the hermetic harness (auto-included).
