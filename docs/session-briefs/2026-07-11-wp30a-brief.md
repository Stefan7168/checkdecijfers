## What WP30a is

Give the pipeline a SOURCE IDENTITY without changing a single observable byte: a code-level
source registry, `Attribution.source`, the D3 display consolidation (as amended), and the D4
migration as a FILE. Every existing answer, citation, chart spec, CSV, stored audit row and LLM
request stays byte-identical — `source: 'cbs'` (or ABSENT, per A1) resolves to today's exact
strings. WP30b (conformance harness + guide) is a SEPARATE later session (A5). WP30c (real
second source + public-claim wording) stays owner-deferred (#123).
