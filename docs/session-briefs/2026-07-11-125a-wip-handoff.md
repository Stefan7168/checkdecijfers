### Done in the WIP

- **`docs/decisions/031-unit-expansion-derivation.md`** — the full ADR: decision D1–D7, ≥4 real
  alternatives (incl. why the LLM must NOT render the expansion, and why the owner rejected the
  "duizend" rewording), trade-offs, revisit triggers. **The design is complete and is the
  authoritative spec for finishing the build.** Read it first.
- **`src/query/types.ts`** — `DerivationRecord` gains a `'unit_expansion'` union member
  (`factor: number`, `value: number`, `unit: 'aantal'`, never `explicit`).
- **`src/query/derivations.ts`** — two new functions:
  - `parseFactorUnit(unit)` — matches PURE numeric factor units only (`x 1 000`, `x 1000`,
    `× 1.000`); any letter in the unit excludes it, so `1 000 euro` and rate units like
    `aantal per 1 000 inwoners` can never match. Returns the factor as a safe integer ≥ 10, else null.
  - `deriveUnitExpansion(cell)` — EXACT integer-scaled arithmetic (never float `value*factor`,
    which is inexact: `390.2*1000 = 390200.00000000006`). Scales by `10^decimals`, verifies exact
    round-trip, requires a safe-integer integer-valued expansion, else refuses. Refuses null cells.
- **`src/query/run.ts`** — imports `deriveUnitExpansion`; pre-registers one record per non-null
  cell (all shapes) after the comparison-max block. A refusal is simply omitted (fail-open).
- **`src/answer/compose/prompt.ts`** — `buildPhrasingPayload` **filters `unit_expansion` out of
  the serialized derivations**. This is load-bearing: it keeps every LLM request byte-identical
  to the pre-#125a form, so **every recorded fixture keeps its request hash and replays — no
  re-record, no live LLM spend** (verify this claim holds; it's the whole reason the design is
  hermetic).
- **`src/answer/compose/validate.ts`** — `derivationNumbers` returns `[d.value]` for the new
  kind (so the expanded token classifies as `derivation`, never `unbacked`); `derivationSourceCells`
  maps it to `sourceResultIds`; **`UNIT_SUFFIX` is now exported** for the splice module to reuse.
