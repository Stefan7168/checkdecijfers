## 1. nederlandinbeeld.org (NiB) — deep dive

**What it is.** Migration-focused data platform by an anonymous "burgerinitiatief": ~80 topic pages (asiel, wonen,
criminaliteit, kosten), tools (Data-Assistent chat, beleidsimpact-calculator, correlaties, mediaframing), petition/protest
pages. **Not neutral**: explicitly migration-critical framing; headline "€46,1 mld migratiekosten" is their OWN bottom-up
gross calculation (own methodology page admits taxes/contributions of migrants are NOT subtracted), presented alongside
real CBS figures.

**Stack (verified from shipped bundles + their own /technische-opbouw page).** Lovable + Cursor + Claude built; Vite +
React + TypeScript + Tailwind + shadcn/ui charts (Recharts) + Leaflet maps; Supabase (DB with RLS + Edge Functions);
457 code-split chunks. Data model is hybrid: **finalized historical series are baked into the frontend bundle as static
JS** ("status: definitief" objects); current-year tiles read Supabase tables (`coa_instroom`, `ind_maandcijfers`,
`rechtspraak_cache`) via React Query (1–5 min staleTime).

**CBS approach.** Scheduled sync (GitHub Actions crons + a **Mac mini at home running an OpenClaw agent 24/7**) → Edge
Functions call CBS StatLine OData v4 (12+ endpoints, 7-day cache); IND/COA are **scraped**; Rechtspraak.nl via API
(2,600+ rulings); 71+ WODC/SCP/IND **PDFs parsed by AI**. Same ground pattern as our principle (b), without our
validation layer. Their claim "cijfers 1:1 doorgegeven zonder AI-interpretatie" is contradicted by their own pipeline.

**The /v1/ask smoking gun.** Their public API's AI endpoint returns prose + key_figures generated by
`google/gemini-3-flash-preview` (visible in the response `meta.model`); sources are document titles ("COA Jaarstukken
2023"), not cells. The LLM produces the numbers — exactly what our principle (a) forbids. Their per-page "xAI
Paginaverificatie" (Grok scores page claims, e.g. "3 van 4 correct") is an LLM checking an LLM, ends with "controleer
bronnen voor definitieve verificatie", and a claim scored **"Niet verifieerbaar" stays published anyway**. Verification
as theater; our refuse-behavior is the real version.

**Correlaties page (owner found it interesting).** 13 hand-picked correlation cards (e.g. "% Islam ↔ migratiekritisch
stemgedrag r=−0.62", "DENK ↔ migratiepercentage r=+0.88") with r-values to two decimals. Methodological problems: pair
selection IS the editorial act (no confounder pairs offered), r over aggregates (ecological fallacy; tiny n; outlier
cities dominate), sources/years/definitions mixed, a before/after comparison dressed as r, no n/CI anywhere. A tab is
named "Veiligelanders" — a coined political term — on a site claiming "alleen CBS-definities". Core lesson:
**traceability ≠ validity of a claimed relationship.**

**Patterns worth copying (the good part).** Source badges per page with CBS table code + LIVE/CACHED status + StatLine
link + "wekelijks bijgewerkt"; event annotation lines in time series ("Syrische crisis", "Oekraïne (RTB)"); definition
toggles ("incl. Oekraïners (RTB)"); per-chart footer with source + share/download-PNG (attribution baked in); an
`llms.txt` (839 lines — aggressively feeds LLMs their narrative incl. how to explain their €46bn method; we would do
this honestly); correctielogboek; databronnen page with sync status; "datablinde vlekken" page (which data does NOT
exist — our principle (c) as a feature); embed routes; a public API + **MCP endpoint** (demand signal for
machine-consumable answers).

## 2. aidscope.co.uk — deep dive

**What it is.** Every pound of UK ODA 2024 (£14.082bn, 5,362 activities, 140 recipients) searchable/mapped; built by an
AI dev shop (SimplyAi Solutions), dataset stamped "prepared 17 July 2026". Editorial angle visible in "Woke" and
"Waste" tabs and bands "acceptable → indefensible".

**Architecture.** **No backend at all**: React SPA + static JSON files (`/data/core.json` etc.) baked from FCDO
accredited official statistics (ODS spreadsheets), numbered source footnotes to gov.uk. A frozen, polished snapshot.

**The surprise: scores are NOT AI.** The "Woke-o-meter" and "Waste-risk" are **fully published deterministic rules**
(trigger vocabulary with weights, title hits +60%, humanitarian damping −45%; waste = additive facts: +52 in-donor
refugee costs, +18 purpose text <60 chars, +14 micro-grants…). Methodology page states plainly: "The editorial scores
are exactly that — editorial. The rules are published above, the inputs are quoted on every activity." Plus a
**reconciliation & limitations section** documenting a £53.6k (0.0004%) headline-vs-rows difference, rounding
tolerances, "benefitting country ≠ cash to that government", and REDACTED rows.

**Lessons.** (1) "Every score shows its workings" = audit-trail as visible UI — strongest external validation yet of
our traceability-as-product bet. (2) Reconciliation/limitations sections build trust — CBS data has the same traps
(suppressed cells, rounding, definition breaks). (3) Deterministic rules make even an *opinion* layer auditable — but
**deterministic ≠ neutral**: the vocabulary choices ARE the politics. Our neutrality = no editorial layer at all.
(4) Static pre-baked JSON is a great pattern for shareable/embed pages. (5) Meta-lesson: polished data sites are now
cheap to build (AI shops produce them in days); the non-commodity part is a continuously refreshed, validated pipeline
with Q&A and guarantees — our part.

## 3. Verdienmodel discussion (owner doubt, taken seriously)

Owner: "open data + every LLM can already answer and chart this — I see our added value evaporating." Assessment
recorded: the doubt is half right — bare "chat over StatLine + chart" IS commoditizing. But **access ≠ answers**: bare
LLMs silently pick definitions (8% vs 56% sociale-huur example — both correct, comparison group differs), compute
themselves, cite stale/news numbers, and don't refuse. The failure mode is silent wrongness, which grows MORE dangerous
as answers get more fluent. Real market = users for whom a wrong number has a cost (journalists, beleid, onderzoek), not
casual askers. The MCP/API flip turns the threat into the market: be the tool other LLMs call. NiB's Grok-verification
and gemini-flash /v1/ask are demand proof without the guarantee — our differentiator is the guarantee, which must be
VISIBLE (badges, §5) to convert.

**Proposed measurement (owner: later):** run our 20-task benchmark against bare frontier chatbots (ChatGPT, Gemini,
Grok; optionally NiB's Data-Assistent), scored on correct number / correct definition / cell-level source / refusal
behavior. Near-perfect scores → revisit verdienmodel; failures → measured proof of moat + marketing material. Recorded
as [#169](../open-questions.md), no date.

## 4. Source candidates assessed this session

**rijksfinancien.nl (owner-proposed).** MinFin official; CC-0; open-data section with Swagger API (two generic calls,
CSV/JSON — dataset-download-shaped, NOT cell-query OData). Datasets: budgettaire tabellen per begrotingsartikel to
regeling level (2015–2026), begrotingsstaten across budget phases (2013–2026), **Financiële Instrumenten = subsidy/loan
recipients with amounts (2017–2025)**, apparaats/ICT/zorg-uitgaven. Journalistic value high (Prinsjesdag/
Verantwoordingsdag peaks; "follow the money"; makes NiB-style budget-sum claims checkable). Fits us extra well because
its trap is definitional: **begroot ≠ uitgegeven** (ontwerp/suppletoir/realisatie) + artikel renumbering across years —
exactly what our clarify/refuse behavior is built for. Medium adapter delta (file-ingest, not StatLine family). Added to
[#123](../open-questions.md) candidates.

**DNB (owner-proposed, long term).** Statistics API since 2024; banks/pensions/insurers/markets; individual-bank
financials and concentration indicators on data.overheid.nl. Caveats: **API access requires eHerkenning → requires KvK
registration, which is deliberately parked until the site is done** → inherently long-term; only ~11 of 230+ datasets
in the API today. Added to the roadmap enrichment list with that note.
